# morphic

Pure-Rust Source 2 `.vtex_c` texture decoder. Built for the vpkmerge GUI's
conflict-modal texture preview so two mods that touch the same texture can be
compared visually without shipping a .NET runtime.

Status: M1-M7, M9, partial M10 (cubemap faces), M11 (PNG/JPEG path) are in.
Morphic decodes RGBA8888, BGRA8888, DXT1, DXT5, ATI1N, ATI2N, BC7, BC6H
(Rgba16F output), and inline PNG_RGBA8888 / PNG_DXT5 / JPEG_DXT5, covering
~86% of Deadlock's `.vtex_c` corpus by count. Cubemap face selection via
`DecodeOptions::face` (0..=5) works for any format; 3D depth slices and
texture arrays are still pending. RGBA16161616F and inline WebP return
`DecodeError::Unimplemented`. The golden harness diffs LDR formats against
a PNG (mae or byte_exact) and HDR formats against a raw `.f32` sibling
using per-channel `abs + rel` tolerance. Real decoder regressions fail the
build; formats still PENDING (RGBA16F, WebP, depth, arrays) print but stay
non-blocking.

## API

```rust
use morphic::{inspect, decode, DecodeOptions};

// Cheap: just reads the binary header in the DATA block.
let info = inspect(&bytes)?;
println!("{:?} {}x{} mips={}", info.format, info.width, info.height, info.mip_count);

// Allocates and decodes mip 0 of slice/face 0.
let img = decode(&bytes)?;
// Or with options:
let img = morphic::decode_at(&bytes, &DecodeOptions { mip: 0, slice: 0, face: 0 })?;
```

`Image::data` is either `Rgba8(Vec<u8>)` for LDR formats or `Rgba16F(Vec<f16>)`
for HDR formats (BC6H, RGBA16161616F). Top-left origin, row-major.

## Layout

| Module               | What it does                                                |
|----------------------|-------------------------------------------------------------|
| `resource/`          | Source 2 container parser (header + block table)           |
| `kv3/`               | Binary KV3 parser (stub; used by future non-texture readers)|
| `texture/`           | Texture binary header parse                                 |
| `texture/format.rs`  | `TextureFormat` enum; names match VRF's `VTexFormat`        |
| `texture/decode/`    | Per-format pixel decoders                                   |

`resource` and `kv3` are `pub(crate)` for now. Promoted to `pub` when a second
caller (Grimoire) needs raw access.

## Fixtures and goldens

Each `.vtex_c` under `fixtures/` ships with a sibling `.png` (oracle output)
and `.meta.json` (format, dims, source SHA, tolerance). HDR fixtures (BC6H
and any future RGBA16F) also ship a `.f32` sibling so the Rust harness can
diff in float space; the `.png` for HDR is tone-mapped and kept only for
human inspection. The Rust harness diffs morphic's decode against whichever
sibling matches the tolerance kind.

Goldens are generated by the C# oracle at `tools/morphic-oracle/`. From the
workspace root:

```bash
just goldens       # regenerate sibling .png + .meta.json
just test          # cargo test -p morphic
just               # both, in order
```

CI runs `cargo test -p morphic` against the committed goldens. No .NET SDK
is needed for `cargo test`. The harness has three outcomes per fixture:

- `[pass]`: morphic's decode matched the oracle's PNG under the declared tolerance
- `[PENDING]`: format's decoder hasn't landed yet (returns `Unimplemented`); printed but doesn't fail CI
- `[FAIL]`: a real decoder regression: wrong dims, wrong format, bytes off; **fails the build**

`cargo test -p morphic -- --nocapture` shows the full breakdown each run.

See `fixtures/README.md` for the per-fixture provenance log.

## Milestones

| #     | Format          | Status |
|-------|-----------------|--------|
| M0    | Format survey   | Done (`tools/format-counts.csv`) |
| M1    | Container + header parse | Done |
| M2    | RGBA8888 + BGRA8888 decode | Done |
| M3    | DXT1 (BC1)      | Done (via `bcdec_rs`) |
| M4    | DXT5 (BC3)      | Done (via `bcdec_rs`) |
| M5    | ATI1N / ATI2N (BC4/BC5) | Done (via `bcdec_rs`) |
| M6    | BC7             | Done (via `bcdec_rs`) |
| M7    | BC6H            | Done (`bcdec_rs::bc6h_half` unsigned; oracle dumps `.f32`, harness does `hdr_eps` diff) |
| M8    | RGBA16161616F   | Pending (HDR uncompressed; reuses M7's `.f32` oracle + diff path) |
| M9    | Mip 0 selection | Done (slices from end of pixel data) |
| M10   | Cubemaps / arrays / 3D | Partial: all 6 cubemap faces selectable via `DecodeOptions::face` (0..=5). 3D depth slices and texture arrays still pending |
| M11   | Inline PNG / JPEG / WebP | Partial (PNG + JPEG OK; WebP pending) |

`bcdec_rs` covers M3-M7 (BC1, BC2, BC3, BC4, BC5, BC6H, BC7) under one safe
pure-Rust dependency. M7's decoder uses the unsigned BC6H variant (RGB f16
expanded to RGBA with alpha = 1.0). For HDR fixtures the oracle dumps a
`.f32` sibling (raw `RgbaF32` bytes, little-endian, ActualW × ActualH × 16
bytes) and the harness diffs the decoder's f16 output against it in float
space using a per-channel `|a - e| <= abs OR |a - e| <= rel * |e|` rule.
